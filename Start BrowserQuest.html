This page is a launcher. For playable no-install mode, open standalone.html. The original BrowserQuest app in this repository needs a live Node.js + Socket.IO server and browser requests that are commonly blocked when you open files directly from file://.

Open standalone.html Open standalone_info.md Open README Open original BrowserQuest client (advanced)

What works here

  • This launcher opens with one obvious click from the repository folder.
  • It works offline and does not need admin rights, Git, Node.js, Python, or a terminal.
  • It explains the real app blockers, dependencies, and user vs. developer workflows.
  • It includes local BrowserQuest screenshots and links to the real client files.

What does not work here

  • The full multiplayer game cannot run entirely from a clicked local HTML file.
  • No bundled Node.js runtime or auto-starting local server is included in this repository.
  • If you open client/index.html directly, browsers usually block the local worker/XHR setup that the game client expects.

About / Help

Why the full game is blocked in local no-install mode
  • The real app is a client/server multiplayer game, not a self-contained static page.
  • The browser client expects a Socket.IO server on 127.0.0.1:8000 or another configured host.
  • The client also uses RequireJS text loading, Ajax, and a web worker for map data, which are unreliable or blocked in normal file:// mode.
Normal user path

For no-install play, open standalone.html. Then read standalone_info.md for limitations and troubleshooting.

Developer path

Developers can still use npm install, node server/js/main.js, and the existing build flow. See the README for the separated developer instructions.
