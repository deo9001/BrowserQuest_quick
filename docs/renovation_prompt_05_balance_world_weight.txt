# Premium Balance, Power Progression, World Weight, Resource Durability, and Area Expansion Renovation Prompt

================================================================================
PURPOSE OF THIS PROMPT
================================================================================

This is a reusable renovation prompt for transforming an existing HTML /
JavaScript / CSS game into a better-balanced adventure where combat, mining,
enemy strength, player progression, area danger, rewards, and world expansion
feel intentional and satisfying.

This prompt is intended for games that already have playable mechanics, regions,
enemies, player stats, resources, and possibly story/NPC systems, but need a
deeper power curve and stronger feeling of progression.

The goal is to improve the game’s mechanical weight:

- combat balance
- enemy health and damage scaling
- player health, attack, defense, and survivability
- XP and level progression
- enemy tiers and threat ratings
- boss durability
- rare resource durability
- mining/object health and hardness
- area difficulty ramps
- reward pacing
- difficulty communication
- world expansion
- late-game danger
- climactic end-boss progression
- safe spawn and interior reliability
- better NPC dialogue presentation
- optional randomized areas for replayability

This prompt is not primarily about cosmetic polish. It is about making the game
feel better to play: harder where it should be hard, easier where it should be
easy, rewarding when the player grows stronger, and dangerous enough that the
player sometimes needs to retreat and prepare.

================================================================================

## Mission

You are an expert gameplay systems designer, RPG balance designer, progression
designer, combat designer, world designer, JavaScript engineer, and creative
technologist.

Your mission is to renovate an existing HTML / JavaScript / CSS game so that it
has a more sophisticated and satisfying balance model.

The game should no longer feel like enemies die too quickly, the player is too
safe, leveling happens too easily, or rare resources break with the same effort
as common objects.

The player should be able to:

- defeat low-level enemies quickly
- recognize when an enemy or area is more dangerous
- encounter enemies that survive longer because their HP, defense, level, tier,
  and area difficulty justify it
- take meaningful damage in dangerous areas
- feel the need to retreat from areas that are currently too hard
- become stronger through leveling, equipment, upgrades, relics, or preparation
- return later and feel the difference after improving
- mine common resources easily
- work harder to mine rare resources
- understand that rare materials require better tools, more hits, or more power
- progress through increasingly difficult regions
- eventually face a climactic late-game or end-boss challenge
- enter and exit interiors safely
- understand NPC dialogue clearly
- replay optional areas that change between new games

The goal is not to make the game unfair. The goal is to make the game feel
weighted, tuned, dangerous, rewarding, and intentional.

================================================================================

## Non-Negotiable Philosophy

1. Balance must be systemic.
   Do not balance the game by saying “this enemy takes exactly 5 hits.” Enemies
   should survive based on health, defense, level, tier, resistance, area
   difficulty, player attack, player upgrades, and damage formulas.

2. Low-level content should remain approachable.
   Early enemies and starter areas should feel light, readable, and beatable.

3. Hard areas must feel hard.
   Later areas should create pressure. The player should sometimes realize:
   “I am not ready for this yet.”

4. Player power must be earned.
   The player should not level too quickly, gain too much health too early, or
   trivialize enemies without earning upgrades.

5. Enemy power must scale meaningfully.
   More dangerous enemies should have more HP, stronger attacks, better defense,
   special behavior, or stronger area modifiers.

6. Mining and resource gathering must have weight.
   Rare objects should be harder to mine than common objects. Durability,
   hardness, tool power, rarity, and region difficulty should matter.

7. Rewards must justify effort.
   Harder enemies, dangerous areas, rare resources, and bosses should reward the
   player appropriately without causing runaway progression.

8. Area difficulty must be communicated.
   The player should be warned through map labels, enemy names, NPC dialogue,
   signs, danger colors, music, environment, or UI hints.

9. Interior and spawn safety is mandatory.
   The player must never start or transition into a wall, enclosed trap, blocked
   door, doorless building, enemy cluster, or impossible-to-exit interior.

10. Randomness must be intentional and visible.
    Optional randomized areas should show a seed, rune, glyph, symbol, or map
    marker so the player understands the area changed by design, not because of
    a bug.

5.2 Damage Formula
================================================================================

Implement or improve a formula where damage depends on attacker power, defender
defense, level difference, enemy tier, area danger, and optional variance.

Example direction:

```js
function calculateDamage(attacker, defender, context) {
  const attack = attacker.attack || 1;
  const defense = defender.defense || 0;
  const levelDelta = (attacker.level || 1) - (defender.level || 1);
  const levelFactor = 1 + Math.max(-0.35, Math.min(0.35, levelDelta * 0.04));
  const areaFactor = context.areaDamageFactor || 1;
  const variance = 0.9 + Math.random() * 0.2;

  const raw = attack * levelFactor * areaFactor * variance;
  const mitigated = raw - defense * 0.55;

  return Math.max(context.minimumDamage || 1, Math.round(mitigated));
}
```

The exact implementation should fit the existing codebase.

Important:

- Do not let defense reduce damage to zero except in explicit immunity cases.
- Do not let low-level enemies become impossible.
- Do not let high-level enemies die instantly.
- Do not let player scaling trivialize bosses too early.

function xpForNextLevel(level) {

Tune after testing.

- mining skill or level
- resource hardness
- area modifier
- upgrade bonuses

If the player’s tool is too weak, show useful feedback:

5.8 Safe Spawn and Interior Reliability
================================================================================

Fix all spawn and interior trap bugs.

Requirements:

- New game must start in a safe early area.
- Player must not start in Shadow Fortress.
- Player must not start inside an enclosed building with no door.
- Every enclosed structure must have a visible usable exit or be marked
  decorative/locked/ruined.
- Every portal must validate the destination.
- Every load must validate the saved player position.
- If saved position is invalid, move the player to the nearest safe fallback.

Implement safe spawn validation:

```js
function findSafeSpawn(region, preferredPosition, options = {}) {
  if (isSafeSpawn(region, preferredPosition, options)) {
    return preferredPosition;
  }

  const fallback = searchNearbySafeTile(region, preferredPosition, options.radius || 8);
  if (fallback) {
    return fallback;
  }

  return region.defaultSafeSpawn;
}
```

A safe spawn must not overlap:

- wall
- doorframe collision
- closed gate
- enemy
- trap
- solid object
- map boundary
- blocked interior border

5.10 Randomized Optional Areas
================================================================================

Add replayability to non-critical areas.

Requirements:

- Story-critical areas remain static.
- Optional areas can randomize between new games.
- Randomization must be seeded.
- Seed persists in the current save.
- New game generates a new seed.
- Map shows a changing symbol/glyph/rune for randomized areas.

Example:

```js

  randomized: true,
  seed: 842193,
  seedGlyph: "Spiral Moon",

```

Randomizable content may include:

- enemy placement
- resource patches
- chest positions
- decorations
- optional paths
- side objectives
- minor ruins
- non-critical loot

Do not randomize:

- required story NPCs
- required quest objectives
- required exits
- main progression portals
- final boss access
- mandatory keys unless guaranteed reachable
